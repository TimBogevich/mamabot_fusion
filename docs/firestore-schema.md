# Схема коллекции `users`

## Обзор

Коллекция `users` — основной реестр пользователей бота MamaBot. Каждый документ представляет одного пользователя, взаимодействующего с ботом. Документ создаётся при первой команде `/start` и обновляется при изменении профиля или настроек пользователя.

Коллекция служит фундаментом для всех остальных коллекций проекта (`pregnancy_data`, `mood_logs`, `nutrition_logs`), каждая из которых ссылается на пользователя через `chatId`.

---

## Идентификатор документа

```
String(chatId)
```

**Почему:** Telegram `chat.id` — уникальный идентификатор чата пользователя с ботом. Использование `String(chatId)` в качестве идентификатора документа гарантирует:
- Естественную уникальность — один документ на пользователя.
- Быстрый поиск по `chatId` без необходимости в составном индексе.
- Удобную интеграцию с Telegram API — все запросы от бота уже содержат `chatId`.

---

## Справочник полей

| Поле | Тип | Обязательное | По умолчанию | Описание |
|------|-----|-------------|-------------|----------|
| `chatId` | `number` | Да | — | Telegram chat ID. Используется как идентификатор документа. |
| `userId` | `string` | Да | — | Telegram user ID из `update.message.from.id`. |
| `firstName` | `string` | Да | — | Имя пользователя в Telegram. |
| `lastName` | `string` | Нет | — | Фамилия пользователя в Telegram (опционально). |
| `username` | `string` | Нет | — | @username пользователя в Telegram (опционально). |
| `language` | `'ru' \| 'en'` | Да | `'ru'` | Выбранный язык интерфейса. |
| `lmpDate` | `string` | Нет | — | Дата первого дня последней менструации в формате ISO (`YYYY-MM-DD`). |
| `currentWeek` | `number` | Нет | — | Вычисленная текущая неделя беременности (1–42). |
| `partnerCode` | `string` | Нет | — | 6-символьный код-приглашение для связи с партнёром. |
| `role` | `'mom' \| 'partner'` | Да | `'mom'` | Роль пользователя: будущая мама или партнёр. |
| `createdAt` | `Timestamp` | Да | `serverTimestamp()` | Время создания документа (устанавливается Firestore). |
| `updatedAt` | `Timestamp` | Да | `serverTimestamp()` | Время последнего обновления документа (устанавливается Firestore). |

---

## Индексы

Коллекция `users` не требует дополнительных составных индексов:

- **`chatId`** — является идентификатором документа, поэтому индексирован автоматически.
- Все остальные запросы выполняются по `chatId` (чтение, запись, обновление) и используют прямой доступ по документу.

Если в будущем потребуется поиск по `partnerCode` или другим полям, потребуется создать соответствующий составной индекс в Firebase Console.

```json
{
  "chatId": 123456789,
  "userId": "123456789",
  "firstName": "Анна",
  "lastName": "Иванова",
  "username": "anna_ivanova",
  "language": "ru",
  "lmpDate": "2026-01-15",
  "currentWeek": 21,
  "partnerCode": "ABC123",
  "role": "mom",
  "createdAt": "2026-06-14T10:30:00.000Z",
  "updatedAt": "2026-06-14T10:30:00.000Z"
}
```

## Сценарии доступа

| Сценарий | Операция | Код |
|----------|----------|-----|
| Регистрация пользователя | Создание документа | `createUser(chatId, data)` |
| Загрузка профиля при старте | Чтение документа | `getUser(chatId)` |
| Смена языка | Обновление поля `language` | `updateUser(chatId, { language: 'en' })` |
| Установка даты LMP | Обновление поля `lmpDate` | `updateUser(chatId, { lmpDate: '2026-03-01' })` |
| Обновление роли | Обновление поля `role` | `updateUser(chatId, { role: 'partner' })` |
| Создание кода партнёра | Обновление поля `partnerCode` | `updateUser(chatId, { partnerCode: 'XYZ789' })` |

---

## Жизненный цикл документа

1. **Создание** — при первом взаимодействии с ботом (команда `/start`). Заполняются `chatId`, `userId`, `firstName`, `lastName`, `username`, `language`, `role`.
2. **Обновление** — при изменении настроек или прогресса беременности: `language`, `lmpDate`, `currentWeek`, `partnerCode`.
3. **Удаление** — не предусмотрено в рамках текущего функционала. Пользователь может удалить данные через запрос в поддержку.
