# i18n / Локализация — MamaBot

> Дата: 2026-06-15
> Модуль: `functions/src/i18n.js`
> Locale-файлы: `functions/src/locales/{ru,en}.json`

## Обзор

Модуль интернационализации (i18n) обеспечивает поддержку русского и английского
языков в боте MamaBot. Локализованные строки хранятся в JSON-файлах и
загружаются синхронно при первом импорте модуля (кэшируются Node.js `require`).

## Подключение

```js
const { t, setLanguage } = require('./src/i18n');
```

## API

### `t(userId, key, vars?)`

Асинхронно разрешает dot-нотацию ключа в локализованную строку для указанного
пользователя.

| Параметр | Тип | Описание |
|----------|-----|----------|
| `userId` | `number\|string` | Telegram chat ID (используется как ID документа в Firestore) |
| `key` | `string` | Ключ в dot-нотации, например `'menu.week_info'` |
| `vars` | `Record<string, string\|number>` | Опциональные подстановки для `{{placeholder}}` |

**Возвращает:** `Promise<string>`

**Цепочка fallback'ов:**

1. Язык пользователя (из Firestore `users/{userId}.language`)
2. Русский (`'ru'`), если у пользователя нет предпочтения языка
3. Сырой ключ (`key`), если ключ отсутствует в обоих locale-файлах

**Примеры:**

```js
// Простое разрешение ключа
const msg = await t(chatId, 'menu.week_info');
// → "📋 Информация о неделе" (ru) или "📋 Week Info" (en)

// С подстановкой переменных
const msg = await t(chatId, 'onboarding.welcome', { name: 'Анна' });
// → "👋 Добро пожаловать в MamaBot, Анна!..."

// Множественные подстановки
const msg = await t(chatId, 'onboarding.week_calculated', {
  week: '5',
  edc: '2026-12-01',
});
// → "🤰 Ты на 5 неделе беременности! Предполагаемая дата родов: 2026-12-01."

// Fallback для отсутствующего ключа
const msg = await t(chatId, 'nonexistent.key');
// → 'nonexistent.key' (без ошибки)

// Нестроковый ключ
const msg = await t(chatId, 123);
// → '123'
```

### `setLanguage(userId, lang)`

Устанавливает язык пользователя в Firestore.

| Параметр | Тип | Описание |
|----------|-----|----------|
| `userId` | `number\|string` | Telegram chat ID |
| `lang` | `'ru'\|'en'` | Код языка |

**Возвращает:** `Promise<string>` — установленный код языка

**Выбрасывает:** `Error`, если `lang` не `'ru'` и не `'en'`

```js
// Установить английский
await setLanguage(chatId, 'en');

// Установить русский
await setLanguage(chatId, 'ru');

// Ошибка при неверном коде
await setLanguage(chatId, 'fr'); // throws Error
```

## Структура locale-файлов

Файлы находятся в `functions/src/locales/`:

```
functions/src/locales/
├── ru.json    # Русские строки
└── en.json    # Английские строки (идентичная структура ключей)
```

### Текущие ключи (контракт)

```
onboarding
├── welcome              — приветствие (может содержать {{name}})
├── choose_language      — выбор языка
├── language_ru          — текст кнопки «🇷🇺 Русский» (одинаков в обоих файлах)
├── language_en          — текст кнопки «🇬🇧 English» (одинаков в обоих файлах)
├── language_saved       — подтверждение выбора языка с {{lang}}
├── already_registered   — сообщение для повторного /start
├── ask_lmp              — запрос даты ПМ (формат ДД.ММ.ГГГГ)
├── lmp_format_hint      — подсказка формата даты (ДД.ММ.ГГГГ)
├── invalid_date         — ошибка неверной даты (формат ДД.ММ.ГГГГ)
├── date_in_future       — ошибка даты в будущем
├── date_too_old         — ошибка: дата старше 42 недель
├── week_calculated      — подтверждение с {{week}} и {{edc}}
├── edd_confirm           — сообщение с авто-ПДР и кнопками Верно/Исправить (с {{edd}})
├── edd_correct           — текст кнопки «✅ Верно» (callback: onboarding_confirm_edd)
├── edd_edit              — текст кнопки «✏️ Исправить» (callback: onboarding_edit_edd)
└── edd_confirmed         — финальное подтверждение после сохранения ПДР (с {{edd}})

menu
├── week_info            — информация о неделе
├── mood_diary           — дневник настроения
├── nutrition            — питание
├── settings             — настройки
├── invite_partner       — пригласить партнёра
├── back                 — назад
└── my_week              — моя неделя

> **Примечание о формате даты:** Начиная с задачи FN-025, ключи `ask_lmp`,
> `lmp_format_hint` и `invalid_date` используют формат **ДД.ММ.ГГГГ**
> (ранее использовался ГГГГ-ММ-ДД). Пользователь вводит дату через точку,
> например `15.03.2026`. Внутренне дата сохраняется в ISO-формате YYYY-MM-DD.

### Соответствие ключей `menu.*` кнопкам главного меню

Кнопки главного меню формируются модулем `functions/src/handlers/menu/mainMenu.js`
и используют следующие ключи для локализованных подписей:

| Ключ               | Подпись кнопки         | callback_data      |
|--------------------|------------------------|--------------------|
| `menu.my_week`     | «📋 Моя неделя»        | `menu_my_week`     |
| `menu.mood_diary`  | «😊 Дневник настроения»| `menu_mood_diary`  |
| `menu.nutrition`   | «🍎 Питание»           | `menu_nutrition`   |
| `menu.settings`    | «⚙️ Настройки»         | `menu_settings`    |

settings
├── title                — заголовок настроек
├── change_language      — смена языка
├── view_lmp             — просмотр даты ПМ
├── reset_data           — сброс данных
└── reset_confirm        — подтверждение сброса

error
├── unknown_callback     — неизвестная команда. Возвращается callback-роутером
│                        при неизвестном callback_data. После сообщения
│                        роутер вызывает showMainMenu() для возврата в главное меню.
├── generic              — общая ошибка
└── session_expired      — сессия истекла
```

### Добавление новых ключей

1. Добавьте ключ в оба файла (`ru.json` и `en.json`) с идентичной структурой
2. Используйте вложенность объектов для группировки (например,
   `"menu": { "back": "🔙 Назад" }`)
3. Значения должны быть непустыми строками
4. Вызов в коде: `await t(userId, 'menu.back')`

## Подстановка переменных

Шаблон `{{variable}}` в строке заменяется на соответствующее значение из
объекта `vars`. Незаданные переменные остаются нетронутыми (без ошибки).

```js
// Ключ: "welcome": "Привет, {{name}}!"
await t(userId, 'welcome', { name: 'Мария' });
// → "Привет, Мария!"

// Ключ: "week_calculated": "Ты на {{week}} неделе!"
await t(userId, 'week_calculated', {});  // пустой vars
// → "Ты на {{week}} неделе!" (плейсхолдер остаётся)
```

## Интеграция с Firestore

Модуль использует Firestore для хранения языковых предпочтений пользователей:

- **Чтение:** `getUser(userId)` — достаёт документ из `users/{chatId}`
- **Запись:** `updateUser(userId, { language: lang })` — обновляет поле `language`

Поле `language` в Firestore имеет тип `'ru' | 'en'`.
